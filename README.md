## рҹ”¬ Core Engine Breakdown

### Stage 1 вҖ” Structured Fact Extraction
> `src/lib/engines/structured-extractor.ts`

Converts unstructured natural language startup descriptions into a rigorous typed JSON schema вҖ” extracting customer personas, market geography, pricing models, competitive assumptions, and validation signals using AI with deterministic post-processing.

### Stage 2 вҖ” Venture Knowledge Graph
> `src/lib/engines/knowledge-graph.ts`

Maps entities (Customer, Problem, Product, Competitor, Revenue Model) into a directed knowledge graph with typed edges. Enables downstream engines to traverse the business model and detect logical gaps without ever re-parsing the original text.

### Stage 3 вҖ” Deterministic Rule Engine
> `src/lib/engines/rule-engine.ts`

Executes **16 hard-coded logic rules** against the Knowledge Graph. Examples:

| Rule ID | Check | Severity |
|---------|-------|----------|
| `R-01` | B2C target with enterprise pricing | рҹ”ҙ Critical |
| `R-04` | Marketplace model with no supply-side | рҹ”ҙ Critical |
| `R-07` | No stated competitive differentiation | рҹҹЎ Warning |
| `R-11` | TAM < $100M with VC exit assumptions | рҹ”ҙ Critical |
| `R-14` | Single-channel GTM dependency | рҹҹЎ Warning |
| `R-16` | Team has zero domain expertise | рҹ”ҙ Critical |

### Stage 4 вҖ” Heuristic Scoring Engine
> `src/lib/engines/scoring-engine.ts`

Calculates scores across **10 venture dimensions** using weighted mathematical formulas, not AI guesses:

```
Overall Score = ОЈ(dimension_score Г— weight)

Dimensions:
  Problem Clarity        вҶ’  15%
  Customer Definition    вҶ’  12%
  Market Size (TAM)      вҶ’  15%
  Business Model         вҶ’  15%
  Revenue Feasibility    вҶ’  10%
  Competitive Moat       вҶ’  12%
  Team Fit               вҶ’  10%
  GTM Strategy           вҶ’   6%
  Validation Evidence    вҶ’   5%
```

### Stage 5 вҖ” Evidence Engine
> `src/lib/engines/evidence-engine.ts`

Compiles verified supporting evidence vs. identified missing evidence. Produces a completeness index that informs the confidence level on every score dimension.

### Stage 6 вҖ” Consistency Engine
> `src/lib/engines/consistency-engine.ts`

Cross-references user questionnaire claims against Tavily external competitor research results. Flags specific contradictions with their source, evidence text, and explanation вҖ” displayed as amber warning cards in the UI.

### Stage 7 вҖ” External Research (Tavily)
> `src/lib/engines/external-research.ts`

Runs Tavily API searches to map the real competitor landscape. Returns structured competitor profiles with name, market position, funding status, and weakness analysis.

### Stage 8 вҖ” AI Explainer (NVIDIA NIM вҶ’ Gemini Fallback)
> `src/lib/engines/ai-explainer.ts`

After all deterministic engines complete, generates human-readable narrative outputs вҖ” SWOT analysis, GTM strategy, MVP roadmap, elevator pitch, investor narrative, and landing page copy вҖ” strictly grounded in the deterministic results.

### Stage 9 вҖ” AI Cross-Verification
> `src/lib/engines/ai-explainer.ts` вҶ’ `generateCrossVerification()`

An independent LLM review that produces its own venture rating (0-100) without seeing the deterministic score. The **Agreement Score** = `100 - |AI_score - deterministic_score|`. Results above 90% agreement are marked `вң“ Very High Agreement`.

## рҹҡҖ Getting Started

### Prerequisites

- Node.js 18+ and npm
- A [Supabase](https://supabase.com) project (free tier works)
- A [Tavily](https://tavily.com) API key (free tier available)
- An [NVIDIA NIM](https://build.nvidia.com) API key **or** a [Google Gemini](https://aistudio.google.com) API key

### 1. Clone the repository

```bash
git clone https://github.com/dathasaiswaroopgudimella-png/Venturelens.git
cd Venturelens
```

### 2. Install dependencies

```bash
npm install
```

### 3. Configure environment variables

```bash
cp .env.example .env.local
```

Open `.env.local` and fill in your credentials:

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key

# AI Providers (NVIDIA NIM is primary, Gemini is fallback)
NVIDIA_NIM_API_KEY=nvapi-your_nvidia_key
GEMINI_API_KEY=your_gemini_key

# External Research
TAVILY_API_KEY=tvly-your_tavily_key
```

### 4. Set up Supabase

Run the SQL schema in your Supabase SQL editor:

```bash
# Copy contents of supabase/schema.sql into your Supabase SQL editor
# Or use the Supabase CLI:
supabase db push
```

### 5. Run the development server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) to see VentureLens AI in action.

## рҹ”җ Security & Architecture Principles

- **No hardcoded secrets** вҖ” all API keys stored in `.env.local`, excluded from version control
- **`.env.local` is git-ignored** вҖ” only `.env.example` template is committed
- **Deterministic-first** вҖ” LLMs only generate narrative explanations; all scores are mathematical
- **Audit-ready outputs** вҖ” every score, rule evaluation, and evidence point is traceable to source
- **SSR auth** вҖ” Supabase sessions handled server-side via cookie middleware
- **Zero-dependency charts** вҖ” SVG charts are hand-crafted, no chart library bloat
